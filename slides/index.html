Eclipse ja Tomcat kuntoon

  1. Lataa Eclipse Luna (versio: IDE for Java EE developers)
  2. Lataa Tomcat 7
  3. Pura Eclipsen zip-paketti
  4. Pura Tomcatin zip-paketti
  5. Avaa Eclipse
  6. Lisää Java EE-perspektiivissä Servers-välilehdelle uusi palvelin (mallia Tomcat 7).
    Huom! Tee tämä ennen kuin tuot demoprojektit Eclipseen.
    File -> New -> Server -> Tomcat 7
    Valitse Tomcatin asennushakemistoksi juuri purkamasi tomcat-kansio.

    Esimerkkiprojektit Eclipseen

  1. Fork repo @ Github
    Luo oma Github-tunnus osoitteessa https://github.com/join
    Mene osoitteeseen https://github.com/agileooy/spring ja paina Fork
  2. Tuo demoprojektit Eclipseen
    File -> Import -> Git -> Projects From Git -> Clone URI -> URI: https://github.com/omatunnari/spring.git -> Import existing projects -> Select All -> Finish
  3. Avaa ssh-tunneli tietokantapalvelimelle,
    mikäli sovellus käyttää tietokantaa
    Linux & OS X: ssh -L 3306:localhost:3306 yourusername@dbserveripaddress
    Windows + Putty: http://tiny.cc/puttytunnel
  4. Lisää webbiprojekti Tomcat-palvelimelle
  5. Käynnistä Tomcat
  6. Avaa selain webbiprojektin aloitussivulle
    http://localhost:8080/projectname

Uuden Webbiprojektin luominen

1. Valitse Java EE -perspektiivi
Window -> Open Perspective -> Java EE (default)

2. Luo uusi projekti
File -> New -> Dynamic Web Project
(Target Runtime: Apache Tomcat 7.0)

3. Luo testisivu projektiin
File -> New -> JSP File

Valitse tiedoston sijainniksi projektisi WebContent-hakemisto
Anna tiedostolle nimeksi index.jsp
Kirjoita body-tagien sisään sana "testisivu".
4. Käynnistä projekti palvelimella
[Select Project]
Run -> Run as... -> Run on Server -> Tomcat -> Finish

5. Avaa ulkoinen selain
Kopioi Eclipsen sisäisestä selain-ikkunasta webbisovelluksesi osoite
ulkoisen selaimen osoitekenttään (esim. Chrome) ja lataa sivu
uudestaan ulkoisella selaimella.

Uuden Maven-projektin luominen

New Maven Project->

[x] Create a simple project (skip archetype selection)
Next->

Group Id: fi.yritys.sovellus
Artifact Id: projektin-nimi
Packaging: jar (komentoriviprojekteille), war (webbiprojekteille)
Name: Projektin nimi
Description: Projektin pidempi kuvaus
Finish

Avaa pom.xml
Määrittele kirjastoriippuvuudet
Voit käyttää dependencyjen määrittelemisessä apuna hakukonetta:
http://mvnrepository.com
